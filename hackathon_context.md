# Deep Agents Hackathon — Context & Reference
*Last updated: March 27, 2026*

---

## Hackathon Theme

**Deep Agents Hackathon**
*Build agents that plan, reason, and execute across complex multi-step tasks autonomously.*

Target use cases include:
- Research agents that go 10 layers deep
- Customer support systems that actually resolve issues
- Data pipelines that run themselves
- Copilots that take action, not just suggestions

---

## Context Engineering Challenge (Primary Brief)

> **"Build agents that don't just think, they act"**

Build infrastructure for **autonomous, self-improving AI agents** that can:
- Tap into real-time data sources
- Make sense of what they find
- Take meaningful action **without human intervention**

Your challenge is to showcase your agent's skills and how they **continuously learn and improve as they operate** — creating solutions that feel **alive, adaptive, and built for real-world impact.**

---

## Judging Criteria

| Criterion | Description |
|-----------|-------------|
| **Autonomy** | How well does the agent act on real-time data without manual intervention? |
| **Idea** | Does the solution have the potential to solve a meaningful problem or demonstrate real-world value? |
| **Technical Implementation** | How well was the solution implemented? |
| **Tool Use** | Did the solution effectively use at least 3 sponsor tools? |
| **Presentation (Demo)** | Demonstration of the solution in **3 minutes** |

> ⚠️ **Demo is 3 minutes.** Build the entire pitch around this constraint.

## Strategic Notes

- **Minimum 3 sponsor tools required** for Tool Use criterion — target 5+ for differentiation
- **Demo is exactly 3 minutes** — ruthlessly rehearse and cut
- Judges from Overmind, TrueFoundry, Aerospike, and Macroscope are builders in the agent infrastructure space — they will recognize architectural sophistication
- Autonomy is the first criterion listed — lead with it
- "Continuously learn and improve" is the brief's own language — use it back at them in the pitch
- Real-time data integration is explicitly called out in the Autonomy criterion — live API calls during demo are critical
